## Topics: EC2 — Introduction, Instance Types, Launch, Boot Scripts

### Concepts Covered
- **EC2 (Elastic Compute Cloud):** Virtual servers in AWS — rent compute instead of owning hardware
- **Instance Types:**
  - General Purpose (t3, m5) — balanced CPU/RAM, good for APIs and small pipelines
  - Compute Optimized (c5) — CPU-heavy workloads
  - Memory Optimized (r5) — in-memory processing, large datasets (Spark on EMR)
  - Storage Optimized (i3) — high disk throughput (data warehousing)
- **Launching EC2:** AMI (OS image) → Instance type → Key pair → Security group → Launch
- **Boot Script (User Data):** Shell script runs once on first start —
  install dependencies, start services, pull code automatically

### Real-World Connection
- Data Engineers use EC2 to host Airflow schedulers, run Spark jobs,
  or serve FastAPI pipelines before moving to managed services
- Memory Optimized (r5) is what you'd pick for PySpark jobs outside EMR
- Boot scripts = infrastructure-as-code lite — same idea as Docker ENTRYPOINT
