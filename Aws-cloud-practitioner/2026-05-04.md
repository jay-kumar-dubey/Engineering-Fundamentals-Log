# AWS Log — May 4, 2026

## Topics: IAM, CLI, CloudShell

### Concepts Covered
- **IAM (Identity and Access Management):** Controls who can access what in AWS
- **Users & Roles:** Users = real people/services; Roles = temporary permission sets
- **Permissions:** Attached via policies (JSON documents)
- **Access Keys:** Used for programmatic access (CLI, SDK) — never hardcode these
- **AWS CLI:** Command-line tool to interact with AWS — automates what you'd do in the console
- **CloudShell:** Browser-based terminal inside AWS console — pre-authenticated, no setup needed

### Key Understanding
- IAM is the security backbone of any AWS setup
- Access Keys = credentials for non-human access (scripts, pipelines)
- CLI usage is the real-world standard — console is just for learning

### Real-World Connection
- Every company uses IAM to manage who can deploy, who can read S3, who can touch prod
- Data Engineers use CLI to trigger Glue jobs, push to S3, check CloudWatch logs
- Least privilege principle: give only the permissions a role actually needs

- IAM is a core exam domain — expect policy evaluation questions
