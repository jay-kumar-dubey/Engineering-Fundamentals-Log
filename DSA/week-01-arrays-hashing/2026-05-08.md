## Q5: Top K Frequent Elements (Medium) — Arrays & Hashing
**Problem:** Return k most frequent elements in array.

**Approach:**
- Build frequency HashMap → sort by value descending → slice first k → return keys
- O(n log n) due to sort

**Note:** Didn't check optimal (Bucket Sort O(n) or Heap O(n log k)) — do this tomorrow.

**Code:**
```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        hashmap = {}
        for i in range(len(nums)):
            if nums[i] in hashmap:
                hashmap[nums[i]] += 1
            else:
                hashmap[nums[i]] = 1
        sort_hash = dict(sorted(hashmap.items(), key=lambda x: x[1], reverse=True)[:k])
        return list(sort_hash.keys())
```

**Why This Exists / Where It Lives in the Wild:**
- Top-K is one of the most common DE query patterns —
  top 10 products by sales, top users by activity, top errors by frequency.
  In systems: heap-based Top-K is how Spark and query engines handle
  ORDER BY x LIMIT k efficiently without sorting the full dataset.
