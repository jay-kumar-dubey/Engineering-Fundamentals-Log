## Q6: Products of Array Except Self (Medium) — Arrays & Hashing

**Problem:** Return array where output[i] = product of all elements except nums[i].
Constraint: O(n) time, no division.

---

**Attempt 1 — Brute Force (my solution):**
- Nested loop — for each index, multiply all others skipping current
- O(n²) time — works but fails the follow-up constraint

```python
class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        res = []
        p = 1
        count = 0
        while count < len(nums):
            for i, n in enumerate(nums):
                if i == count:
                    continue
                else:
                    p *= n
            res.append(p)
            count += 1
            p = 1
        return res
```

---

**Optimal — Prefix × Postfix (learned):**
- Pass 1 (left→right): res[i] = product of everything to the LEFT of i
- Pass 2 (right→left): multiply res[i] by product of everything to the RIGHT of i
- Result: res[i] = left product × right product = product of everything except self
- O(n) time, O(1) extra space (output array doesn't count)

```python
class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        res = [1] * len(nums)

        prefix = 1
        for i in range(len(nums)):
            res[i] = prefix
            prefix *= nums[i]

        postfix = 1
        for i in range(len(nums) - 1, -1, -1):
            res[i] *= postfix
            postfix *= nums[i]

        return res
```

**Why This Exists / Where It Lives in the Wild:**
- Division-free prefix product is how columnar databases compute
  running totals without touching already-aggregated rows.
- In DE: same two-pass pattern appears in Spark when computing
  cumulative products across partitions — left pass on worker,
  right pass on driver to combine.
- The insight (split one hard problem into two easy passes) is
  the foundation of scan-based query execution in databases.

**Key Pattern:** Can't do it in one pass? → Split into prefix pass + postfix pass.
